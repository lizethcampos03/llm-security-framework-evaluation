# Dataset Metadata Schema

This document references the official metadata specification for the SecurityEval-derived contextual benchmark.

See:

```text
docs/dataset/02_dataset_schema.md
```

All dataset metadata records should conform to the required schema described in that document.

The purpose of maintaining a centralized schema is to ensure:

- consistency
- reproducibility
- traceability
- experiment compatibility

Future schema revisions should be documented through repository decision records and changelog entries.
