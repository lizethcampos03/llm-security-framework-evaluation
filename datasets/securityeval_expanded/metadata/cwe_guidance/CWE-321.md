# CWE-321 Guidance

## CWE ID

CWE-321

## CWE Name

Use of Hard-coded Cryptographic Key

## Source

MITRE CWE official entry:

https://cwe.mitre.org/data/definitions/321.html

## Weakness Summary

The product uses a hard-coded, unchangeable cryptographic key.

## Mitigation Guidance

- Phase: Architecture and Design - Prevention schemes mirror that of hard-coded password storage.

## Dataset Usage

This guidance will be used when generating secure counterparts for vulnerable samples mapped to CWE-321.

The secure counterpart should preserve intended functionality while removing or mitigating the target weakness using CWE-specific guidance.

## Notes

This file summarizes MITRE CWE guidance for dataset generation and secure counterpart creation.
